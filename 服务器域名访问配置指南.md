# FinFlow 服务器域名访问配置指南

## 🎯 问题描述

部署成功后：
- ✅ `http://127.0.0.1:5273/` 可以访问
- ❌ `http://fn.hyqy.group:5273/` 无法访问

## 🔍 原因分析

前端服务默认绑定 `127.0.0.1`（本地回环地址），只允许本机访问，不允许外部网络访问。

### 代码逻辑
```python
# 原来的逻辑：把 0.0.0.0 转成 127.0.0.1
def resolve_browser_host(host: str) -> str:
    if host in {"", "0.0.0.0", "::"}:
        return "127.0.0.1"  # <-- 问题在这里
    return host
```

## 🔧 解决方案

### 已完成的修复

修改了 `resolve_frontend_service_settings` 函数：

```python
# 后端使用 resolve_browser_host 转换（用于浏览器访问）
backend_host = resolve_browser_host(effective_config.get("APP_HOST", "127.0.0.1"))
# 前端绑定地址使用原始值，支持 0.0.0.0 外部访问
raw_app_host = (effective_config.get("APP_HOST") or "127.0.0.1").strip()
frontend_bind_host = "0.0.0.0" if raw_app_host in {"0.0.0.0", "::", ""} else raw_app_host
```

### 配置要求

确保后端 `.env` 文件中：
```env
APP_HOST=0.0.0.0
APP_PORT=8100
```

## 📋 防火墙配置

如果仍无法访问，检查服务器防火墙：

### Windows 防火墙
```cmd
# 添加入站规则（PowerShell 管理员）
New-NetFirewallRule -DisplayName "FinFlow Frontend" -Direction Inbound -Protocol TCP -LocalPort 5273 -Action Allow
New-NetFirewallRule -DisplayName "FinFlow Backend" -Direction Inbound -Protocol TCP -LocalPort 8100 -Action Allow
```

### 或者通过图形界面
1. 打开"Windows Defender 防火墙"
2. 点击"高级设置"
3. 点击"入站规则" → "新建规则"
4. 选择"端口" → "TCP" → 输入 `5273,8100`
5. 选择"允许连接"
6. 应用规则到所有配置文件
7. 命名为 "FinFlow Services"
