## 各标签页详细字段

### 📋 基本信息 (Basic Info)
| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| API 名称 | 文本 | ✅ | 接口的显示名称，例如"查询用户列表" |
| 请求方法 | 选择 | ✅ | GET, POST, PUT, DELETE, PATCH 之一 |
| 接口路径 | 文本 | ✅ | 相对路径，例如 /api/v1/users |
| 所属凭证服务 | 选择 | ✅ | 此接口使用的外部系统凭证 |
| 简要描述 | 文本 | ❌ | 一句话功能描述 |
| 启用此接口 | 复选框 | ❌ | 控制接口是否在生产中使用 |

### 📤 请求配置 (Request Config)
| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 请求头 | JSON代码 | ❌ | HTTP Headers，支持变量替换 |
| 请求体示例 | JSON代码 | ❌ | 请求体的 JSON 格式示例 |

**常见变量：**
- `{token}`: 访问令牌
- `{API_KEY}`: API 密钥
- `{timestamp}`: 当前时间戳
- `{user_id}`: 用户 ID

### 📥 响应示例 (Response Example)
| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 响应示例 | JSON代码 | ❌ | 成功响应的完整 JSON 结构 |

**标准响应结构（推荐）：**
```json
{
  "success": true,
  "code": 0,
  "message": "操作成功",
  "data": {}
}
```

### 📝 文档说明 (Documentation)
| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 详细文档 | Markdown | ❌ | 支持完整 Markdown 格式的文档 |

**文档包含内容建议：**
- 功能说明和使用场景
- 详细的参数说明
- HTTP 状态码和错误处理
- 版本历史
- 调用限制和性能提示
- 代码示例

## 使用流程

### ✨ 创建新接口
1. 在主列表中点击 **「新增接口」** 按钮
2. 自动打开编辑窗口，标签页在"基本信息"
3. 填写必填项（带红色 * 标记）
4. （可选）切换其他标签页补充详细信息
5. 点击 **「💾 保存接口配置」** 保存

### ✏️ 编辑现有接口
1. 在 API 列表中找到目标接口
2. 点击行尾的编辑按钮（铅笔图标）
3. 修改需要更新的字段
4. 点击 **「💾 保存接口配置」** 保存

### 🔍 快速查看
- 点击接口行的代码按钮（终端图标）可查看调用示例
- 在列表中搜索框输入关键词可快速筛选

## 数据示例

### 完整的 API 配置示例

**基本信息：**
- 名称：获取用户列表
- 方法：GET
- 路径：/api/v1/users
- 服务：金蝶系统
- 描述：获取系统中所有用户的信息列表

**请求配置：**
```json
{
  "Content-Type": "application/json",
  "Authorization": "Bearer {access_token}",
  "X-Request-ID": "{UUID}"
}
```

**响应示例：**
```json
{
  "success": true,
  "code": 0,
  "message": "获取成功",
  "data": {
    "total": 150,
    "users": [
      {
        "id": "U001",
        "name": "张三",
        "email": "zhangsan@example.com",
        "role": "admin",
        "created_at": "2025-12-01T10:00:00Z"
      }
    ]
  }
}
```

**文档说明：**
```markdown
## 获取用户列表 API

### 功能说明
此接口返回系统中已授权的所有用户列表，支持分页和过滤。

### 请求参数
- `page`: 页码，从 1 开始（默认: 1）
- `pageSize`: 每页记录数，最大 100（默认: 20）
- `filter_role`: 按角色过滤，可选值: admin, user, guest

### 响应字段说明
- `total`: 用户总数
- `users[].id`: 用户唯一标识
- `users[].name`: 用户姓名
- `users[].role`: 用户角色

### 错误码
- 400: 请求参数错误
- 401: 未认证（access_token 无效）
- 403: 权限不足（需要 admin 角色）
- 429: 请求过于频繁（已达到速率限制）

### 调用限制
- 频率限制：每分钟 100 次
- 单次返回最大: 100 条记录

### 更新历史
- v1.2 (2026-01-15): 添加 filter_role 参数
- v1.1 (2025-12-01): 初始版本
```

## 常见问题

**Q: 如何在请求头中使用变量？**
A: 在花括号中写入变量名，如 `{"Authorization": "Bearer {token}"}`，系统会在运行时自动替换。

**Q: 响应示例可以有多种格式吗？**
A: 当前只支持 JSON 格式。如果需要多种响应格式，建议在文档说明中描述。

**Q: 文档说明支持哪些 Markdown 语法？**
A: 支持完整的 Markdown，包括标题、列表、代码块、表格、超链接等。

**Q: 可以导入现有的 OpenAPI 配置吗？**
A: 当前不支持直接导入，但可以通过批量导入功能导入 OpenAPI JSON。

## 性能提示

- 大型响应示例（>50KB）建议在实际测试中验证
- 请求头和请求体不做实时 JSON 验证，保存时会检查格式
- 文档支持无限长度，但建议控制在 50KB 以内
