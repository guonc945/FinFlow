# 金蝶云星空接口：查询供应商

> **接口说明**：本接口用于在金蝶云星空系统中查询供应商基础数据，支持分页检索及多种组合过滤条件。

## 1. 接口基本信息

| 属性 | 内容 |
| :--- | :--- |
| **用途说明** | 查询供应商列表 |
| **请求方式** | `POST` |
| **接口路径** | `/v2/basedata/bd_supplier/query` |
| **接口类型** | 基础数据查询 |

---

## 2. 请求参数详述

### 2.1 请求头 (Headers)

| 参数名称 | 必填 | 示例值 | 说明 |
| :--- | :--- | :--- | :--- |
| `Content-Type` | 是 | `application/json` | 内容格式，固定为 JSON |
| `accesstoken` | 是 | `{{accesstoken}}` | 用户令牌（Token 认证方式） |
| `Idempotency-Key` | 否 | `{{requestId}}` | 幂等性标识，防止并发重复调用 |
| `x-acgw-identity` | 是 | `{{identity}}` | API 身份标识，第三方应用启用后颁发 |

### 2.2 请求体 (Body)

| 参数名称 | 类型 | 必填 | 层级 | 说明 |
| :--- | :--- | :--- | :--- | :--- |
| **data** | Object | 否 | 0 | 过滤条件对象 |
| ├─ `start_createtime` | DateTime | 否 | 1 | 开始时间，格式：`yyyy-MM-dd HH:mm:ss` |
| ├─ `end_createtime` | DateTime | 否 | 1 | 结束时间，格式：`yyyy-MM-dd HH:mm:ss` |
| ├─ `id` | Array | 否 | 1 | 供应商内部 ID 数组 |
| ├─ `number` | Array | 否 | 1 | 供应商编码数组 (如：`["S001", "S002"]`) |
| ├─ `createorg_number` | Array | 否 | 1 | 创建组织编码数组 |
| └─ `status` | Array | 否 | 1 | 数据状态，详见 [状态说明](#5-状态说明) |
| **pageSize** | Integer | 是 | 0 | 每页显示的记录数 (默认建议 10) |
| **pageNo** | Integer | 否 | 0 | 当前页码 (从 1 开始) |

---

## 3. 返回结果说明

| 参数名称 | 类型 | 层级 | 说明 |
| :--- | :--- | :--- | :--- |
| **status** | Boolean | 0 | 接口调用状态 (`true`/`false`) |
| **errorCode** | String | 0 | 结果代码，成功固定为 `"0"` |
| **message** | String | 0 | 如果调用失败，返回详细错误提示 |
| **data** | Object | 0 | 数据结果容器 |
| ├─ `pageNo` | Integer | 1 | 当前页码 |
| ├─ `pageSize` | Integer | 1 | 每页记录数 |
| ├─ `totalCount` | String | 1 | 总记录数 (若不返回则显示 `"-1"`) |
| ├─ `lastPage` | Boolean | 1 | 是否为最后一页 |
| └─ **rows** | Array | 1 | 供应商明细数据列表 |
| &nbsp;&nbsp;&nbsp;&nbsp;├─ `id` | Long | 2 | 供应商唯一标识 ID |
| &nbsp;&nbsp;&nbsp;&nbsp;├─ `number` | String | 2 | 供应商编码 |
| &nbsp;&nbsp;&nbsp;&nbsp;├─ `name` | String | 2 | 供应商全称 |
| &nbsp;&nbsp;&nbsp;&nbsp;├─ `status` | String | 2 | 数据状态代码 |
| &nbsp;&nbsp;&nbsp;&nbsp;├─ `enable` | String | 2 | 使用状态 (`1`: 启用, `0`: 禁用) |
| &nbsp;&nbsp;&nbsp;&nbsp;└─ `createorg_name` | String | 2 | 创建组织名称 |

---

## 4. 示例演示

### 4.1 请求报文示例

```json
{
    "data": {
        "start_createtime": "2024-01-01 00:00:00",
        "end_createtime": "2024-12-31 23:59:59",
        "status": ["C"]
    },
    "pageNo": 1,
    "pageSize": 10
}
```

## 5. 状态说明

在 `status` 字段中支持以下枚举值：

| 代码 | 描述 |
| :--- | :--- |
| **A** | 暂存 (Saved) |
| **B** | 已提交 (Submitted) |
| **C** | 已审核 (Audited) |
| **D** | 重新审核 (Re-auditing) |
