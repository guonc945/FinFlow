# FinFlow SQL Server 数据库配置说明

## 配置文件说明

### `.env` 文件（主配置文件）

**位置**: `backend/.env`

这是**唯一的数据库配置文件**，所有数据库连接参数都写在这里。

**示例配置**:
```env
# ================================================
# 数据库配置
# ================================================
DATABASE_URL=
DB_DIALECT=mssql
DB_DRIVER=ODBC Driver 17 for SQL Server
DB_HOST=localhost
DB_PORT=1433
DB_NAME=finflow
DB_USER=admin
DB_PASSWORD=
DB_TRUST_SERVER_CERT=yes
DB_ENCRYPT=yes

# ================================================
# 安全配置
# ================================================
SECRET_KEY=your_secret_key_here
ACCESS_TOKEN_EXPIRE_MINUTES=1440
ENCRYPTION_KEY=
ENCRYPTION_KEY_FILE=

# ================================================
# 马克系统配置
# ================================================
MARKI_USER=your_marki_user
MARKI_PASSWORD=your_marki_password
MARKI_SYSTEM_ID=your_system_id

# ================================================
# 应用配置
# ================================================
APP_HOST=0.0.0.0
APP_PORT=8110
APP_RELOAD=false
ALLOWED_ORIGINS=http://localhost:5273,http://localhost:3000,http://localhost:8110
ALLOW_LAN_ORIGINS=false
```

**配置项说明**:

#### 数据库配置
- `DB_DIALECT`: 数据库类型 (`mssql` 表示 SQL Server)
- `DB_DRIVER`: ODBC 驱动名称
- `DB_HOST`: 数据库服务器地址
- `DB_PORT`: 数据库端口 (SQL Server 默认 1433)
- `DB_NAME`: 数据库名称
- `DB_USER`: 数据库用户名
- `DB_PASSWORD`: 数据库密码
- `DB_TRUST_SERVER_CERT`: 是否信任服务器证书
- `DB_ENCRYPT`: 是否加密连接

#### 安全配置
- `SECRET_KEY`: JWT 密钥
- `ACCESS_TOKEN_EXPIRE_MINUTES`: 访问令牌过期时间（分钟）
- `ENCRYPTION_KEY`: 数据加密密钥
- `ENCRYPTION_KEY_FILE`: 加密密钥文件路径

#### 马克系统配置
- `MARKI_USER`: 马克系统用户名
- `MARKI_PASSWORD`: 马克系统密码
- `MARKI_SYSTEM_ID`: 马克系统ID

#### 应用配置
- `APP_HOST`: 应用服务器地址
- `APP_PORT`: 应用服务器端口
- `APP_RELOAD`: 是否启用自动重载
- `ALLOWED_ORIGINS`: 允许的跨域来源
- `ALLOW_LAN_ORIGINS`: 是否允许局域网来源

## 代码逻辑

### 数据库配置读取

在 `backend/database.py` 中，配置直接从 `.env` 文件读取：

```python
# 直接从环境变量读取数据库配置
DB_HOST = os.getenv("DB_HOST", "localhost").strip()
DB_PORT = os.getenv("DB_PORT", "1433").strip()
DB_NAME = os.getenv("DB_NAME", "finflow").strip()
DB_USER = os.getenv("DB_USER", "admin").strip()
DB_PASSWORD = os.getenv("DB_PASSWORD", "").strip()
```

### 数据库连接配置

```python
def _build_database_url() -> str | URL:
    if DATABASE_URL:
        return DATABASE_URL

    if DB_DIALECT in {"mssql", "sqlserver", "sql_server"}:
        return URL.create(
            "mssql+pyodbc",
            username=DB_USER,
            password=DB_PASSWORD,
            host=DB_HOST,
            port=int(DB_PORT),
            database=DB_NAME,
            query={
                "driver": DB_DRIVER,
                "TrustServerCertificate": os.getenv("DB_TRUST_SERVER_CERT", "yes"),
                "Encrypt": os.getenv("DB_ENCRYPT", "yes"),
            },
        )
    # ... 其他数据库类型
```

## 使用方法

### 修改数据库配置

1. 编辑 `backend/.env` 文件
2. 修改对应的配置项
3. 重启后端服务

**示例**:
```env
# 修改数据库服务器
DB_HOST=192.168.1.100

# 修改数据库端口
DB_PORT=1433

# 修改数据库名称
DB_NAME=finflow_prod

# 修改数据库用户
DB_USER=sa

# 修改数据库密码
DB_PASSWORD=your_password
```

### 验证配置

```bash
# 检查 .env 文件
type backend\.env
```

## 注意事项

1. **所有配置都写在 `.env` 中** - 这是唯一的配置来源
2. **修改配置后重启服务** - 环境变量只在启动时加载
3. **密码安全** - `.env` 文件包含敏感信息，不要提交到版本控制

## FinFlowManager 配置

FinFlowManager 会读取 `.env` 文件中的配置，并在界面上显示和编辑：

- 配置管理页 → 编辑 `.env` 文件
- 数据库备份 → 使用 `.env` 中的配置连接数据库
