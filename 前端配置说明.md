# FinFlow 前端配置说明

## 🆕 新增功能

在服务管理器的"系统配置"页面，新增了"前端配置"部分，可以独立配置前端服务的监听地址和端口。

## 📋 配置项说明

### 前端配置

| 配置项 | 说明 | 默认值 | 示例 |
|--------|------|--------|------|
| `FRONTEND_HOST` | 前端监听地址 | 空（自动使用 APP_HOST） | `0.0.0.0` 或 `127.0.0.1` |
| `FRONTEND_PORT` | 前端监听端口 | `5273` | `5273` 或 `8080` |
| `FRONTEND_ALLOWED_ORIGINS` | 前端允许来源 | 空 | `http://fn.hyqy.group:5273` |

## 🔧 使用场景

### 场景 1：允许外部访问（推荐）

如果需要允许通过域名或外网 IP 访问前端：

```env
# 后端配置
APP_HOST=0.0.0.0
APP_PORT=8100

# 前端配置
FRONTEND_HOST=0.0.0.0
FRONTEND_PORT=5273
FRONTEND_ALLOWED_ORIGINS=http://fn.hyqy.group:5273,http://your-domain.com:5273
```

### 场景 2：仅本地访问

如果只需要本地访问：

```env
# 后端配置
APP_HOST=127.0.0.1
APP_PORT=8100

# 前端配置（留空则自动跟随 APP_HOST）
FRONTEND_HOST=
FRONTEND_PORT=5273
```

### 场景 3：前后端使用不同地址

如果前后端需要绑定不同地址：

```env
# 后端绑定特定内网 IP
APP_HOST=192.168.1.100
APP_PORT=8100

# 前端绑定所有网卡
FRONTEND_HOST=0.0.0.0
FRONTEND_PORT=5273
```

## 🚀 配置步骤

### 步骤 1：打开服务管理器
运行 `FinFlowManager.exe`

### 步骤 2：进入系统配置页面
点击左侧菜单"系统配置"

### 步骤 3：配置前端参数
在"前端配置"部分填写：
- **前端监听地址**: `0.0.0.0`（允许外部访问）或 `127.0.0.1`（仅本地）
- **前端监听端口**: `5273`（默认）
- **前端允许来源**: 可选，用于 CORS 配置

### 步骤 4：保存配置
点击"保存配置"按钮

### 步骤 5：重启前端服务
1. 停止前端服务
2. 重新启动前端服务

## 🌐 防火墙配置

如果配置了 `FRONTEND_HOST=0.0.0.0`，需要开放防火墙端口：

### Windows 防火墙
```cmd
# PowerShell 管理员
New-NetFirewallRule -DisplayName "FinFlow Frontend" -Direction Inbound -Protocol TCP -LocalPort 5273 -Action Allow
```

### 云服务器安全组
| 协议 | 端口 | 授权对象 | 说明 |
|------|------|----------|------|
| TCP | 5273 | 0.0.0.0/0 | 前端服务 |
| TCP | 8100 | 0.0.0.0/0 | 后端 API |

## ✅ 验证配置

### 检查端口监听
```cmd
netstat -ano | findstr 5273
```
应该显示 `0.0.0.0:5273`（允许外部）或 `127.0.0.1:5273`（仅本地）

### 测试访问
- 本地访问：`http://127.0.0.1:5273/`
- 域名访问：`http://fn.hyqy.group:5273/`
- IP 访问：`http://<服务器IP>:5273/`

## 📝 注意事项

1. **FRONTEND_HOST 为空时**：自动使用 `APP_HOST` 的值
2. **FRONTEND_PORT 为空时**：自动使用 `5273` 或 `frontend/.env` 中的 `VITE_PORT`
3. **修改配置后**：需要重启前端服务才能生效
4. **安全考虑**：生产环境建议配置具体的 IP 地址，而不是 `0.0.0.0`

## 🔍 故障排查

### 问题：外部无法访问

1. 检查 `FRONTEND_HOST` 是否为 `0.0.0.0`
2. 检查 Windows 防火墙是否允许 5273 端口
3. 检查云服务器安全组是否开放 5273 端口
4. 检查端口监听地址：`netstat -ano | findstr 5273`

### 问题：端口被占用

1. 更换 `FRONTEND_PORT` 为其他端口（如 8080）
2. 查找占用端口的进程：`netstat -ano | findstr 5273`
3. 结束占用进程或更换端口

### 问题：配置未生效

1. 确认已点击"保存配置"
2. 确认已重启前端服务
3. 检查 `backend\.env` 文件内容是否正确
