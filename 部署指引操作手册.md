## 1. 部署前准备

### 1.1 部署目标

- **服务器系统**：Windows Server 2012 R2 或更高版本
- **部署模式**：FinFlowManager.exe 托盘管理 + Python 后端进程
- **部署位置**：`D:\FinFlow\`

### 1.2 推荐目录结构

```
D:\FinFlow\
├── backend\          # 后端服务
├── frontend\         # 前端应用
├── deploy\           # 部署脚本
├── tools\            # 工具
├── backups\          # 数据库备份目录（建议）
└── logs\             # 日志目录（建议）
```

### 1.3 部署前检查清单

- [ ] 服务器已安装 Python 3.10 x64
- [ ] 服务器已安装 SQL Server ODBC Driver 17 或更高版本
- [ ] 数据库服务（SQL Server）可正常访问
- [ ] 防火墙已放行所需端口（默认 8100）
- [ ] 有管理员权限进行服务配置

## 2. 服务器环境配置

### 2.1 安装 Python 3.10

1. 下载 Python 3.10 x64 安装包
2. 安装时勾选 "Add Python to PATH"
3. 验证安装：
   ```powershell
   python --version
   ```

### 2.2 安装 ODBC 驱动

下载并安装 [ODBC Driver 17 for SQL Server](https://learn.microsoft.com/en-us/sql/connect/odbc/download-odbc-driver-for-sql-server)

验证安装：
```powershell
python -c "import pyodbc; print(pyodbc.drivers())"
```

应看到类似输出：
```
['ODBC Driver 17 for SQL Server', 'SQL Server']
```

### 2.3 创建目录结构

```powershell
# 创建项目目录
New-Item -Path "D:\FinFlow" -ItemType Directory -Force

# 创建子目录
New-Item -Path "D:\FinFlow\backend\logs" -ItemType Directory -Force
New-Item -Path "D:\FinFlow\backups" -ItemType Directory -Force
```

## 3. 代码部署

### 3.1 从 Git 仓库克隆

```powershell
cd D:\
git clone <你的仓库地址> FinFlow
```

### 3.2 或手动部署

将本地打包的项目文件复制到 `D:\FinFlow\`

确保包含以下目录：
```
backend\
frontend\
deploy\
tools\
```

### 3.3 创建后端虚拟环境

```powershell
cd D:\FinFlow\backend
py -3.10 -m venv .venv
.venv\Scripts\python.exe -m pip install --upgrade pip
.venv\Scripts\pip.exe install -r requirements.txt
```

如果需要 SQL Server 报表功能，额外安装：
```powershell
.venv\Scripts\pip.exe install pyodbc
```

## 4. 后端配置

### 4.1 配置文件位置

- **主配置文件**：`D:\FinFlow\backend\.env`
- **示例文件**：`D:\FinFlow\backend\.env.example`
- **密钥文件**：`D:\FinFlow\backend\.encryption.key`

### 4.2 配置文件内容

复制示例文件并修改：

```powershell
cd D:\FinFlow\backend
Copy-Item .env.example .env
```

编辑 `.env` 文件，填写生产配置：

```env
# ================================================
# 数据库配置
# ================================================
DB_DIALECT=mssql
DB_DRIVER=ODBC Driver 17 for SQL Server
DB_HOST=localhost
DB_PORT=1433
DB_NAME=finflow
DB_USER=admin
DB_PASSWORD=your_password
DB_TRUST_SERVER_CERT=yes
DB_ENCRYPT=yes

# ================================================
# 安全配置
# ================================================
SECRET_KEY=your_secret_key_here
ACCESS_TOKEN_EXPIRE_MINUTES=1440
ENCRYPTION_KEY_FILE=backend\.encryption.key

# ================================================
# 马克系统配置
# ================================================
MARKI_USER=your_marki_user
MARKI_PASSWORD=your_marki_password
MARKI_SYSTEM_ID=your_system_id

# ================================================
# 应用配置
# ================================================
APP_HOST=127.0.0.1
APP_PORT=8100
APP_RELOAD=false
ALLOWED_ORIGINS=http://127.0.0.1:8100,http://localhost:8100
ALLOW_LAN_ORIGINS=false
```

### 4.3 配置说明

#### 数据库配置

| 配置项 | 说明 | 示例值 |
|--------|------|--------|
| `DB_DIALECT` | 数据库类型 | `mssql` |
| `DB_DRIVER` | ODBC 驱动名称 | `ODBC Driver 17 for SQL Server` |
| `DB_HOST` | 数据库服务器地址 | `localhost` 或 IP |
| `DB_PORT` | 数据库端口 | `1433` |
| `DB_NAME` | 数据库名称 | `finflow` |
| `DB_USER` | 数据库用户名 | `admin` |
| `DB_PASSWORD` | 数据库密码 | 你的密码 |
| `DB_TRUST_SERVER_CERT` | 是否信任服务器证书 | `yes` |
| `DB_ENCRYPT` | 是否加密连接 | `yes` |

#### 安全配置

| 配置项 | 说明 | 示例值 |
|--------|------|--------|
| `SECRET_KEY` | JWT 密钥 | 随机字符串 |
| `ACCESS_TOKEN_EXPIRE_MINUTES` | 访问令牌过期时间（分钟） | `1440` (24小时) |
| `ENCRYPTION_KEY_FILE` | 加密密钥文件路径 | `backend\.encryption.key` |

#### 马克系统配置

| 配置项 | 说明 | 示例值 |
|--------|------|--------|
| `MARKI_USER` | 马克系统用户名 | 你的用户名 |
| `MARKI_PASSWORD` | 马克系统密码 | 你的密码 |
| `MARKI_SYSTEM_ID` | 马克系统ID | 你的系统ID |

#### 应用配置

| 配置项 | 说明 | 示例值 |
|--------|------|--------|
| `APP_HOST` | 应用服务器地址 | `127.0.0.1` 或 `0.0.0.0` |
| `APP_PORT` | 应用服务器端口 | `8100` |
| `APP_RELOAD` | 是否启用热重载 | `false` (生产环境) |
| `ALLOWED_ORIGINS` | 允许的跨域来源 | 逗号分隔的 URL 列表 |
| `ALLOW_LAN_ORIGINS` | 是否允许局域网来源 | `false` |

### 4.4 生成加密密钥

新环境首次部署：

```powershell
python -c "from cryptography.fernet import Fernet; print(Fernet.generate_key().decode())"
```

将输出结果写入密钥文件：

```powershell
# 生成密钥
$密钥 = python -c "from cryptography.fernet import Fernet; print(Fernet.generate_key().decode())"

# 写入密钥文件
Set-Content -Path "D:\FinFlow\backend\.encryption.key" -Value $密钥
```

旧环境迁移：必须沿用原来的 `.encryption.key` 文件，不要重新生成。

### 4.5 语法检查

```powershell
cd D:\FinFlow\backend
.venv\Scripts\python.exe -m py_compile main.py database.py models.py schemas.py
.venv\Scripts\python.exe -m py_compile utils\auth.py utils\crypto.py
```

## 5. 前端部署

### 5.1 构建前端（在构建机）

**推荐在开发机或构建机完成前端构建**，避免在服务器上安装 Node.js。

```powershell
cd D:\FinFlow\frontend
Copy-Item .env.production.example .env.production
npm ci
npm run build
```

构建产物位于：`frontend\dist\`

### 5.2 部署前端到服务器

将 `frontend\dist\` 整个目录复制到服务器的 `D:\FinFlow\frontend\dist\`

### 5.3 配置前端环境变量

编辑 `frontend\.env.production`：

```env
VITE_API_BASE_URL=/api
```

## 6. 托盘管理器配置

### 6.1 从源码运行

```powershell
cd D:\FinFlow
py -3.10 -m pip install -r deploy\windows\manager_requirements.txt
py -3.10 tools\finflow_manager.py
```

### 6.2 打包为 EXE

```powershell
cd D:\FinFlow
deploy\windows\build_manager_exe.bat
```

生成文件：`D:\FinFlow\deploy\windows\dist\FinFlowManager.exe`

### 6.3 首次启动配置

1. 启动 FinFlowManager
2. 在"配置管理"页保存 `backend\.env`
3. 确认 `backend\.encryption.key` 已存在
4. 在"服务状态"页点击"启动后端"
5. 点击"打开前端"确认系统可访问
6. 根据需要勾选"Windows 登录后自动启动管理器"

### 6.4 配置管理器功能

#### 配置管理

- 编辑 `backend\.env` 文件
- 保存后自动应用配置

#### 服务控制

- 启动/停止/重启后端服务
- 查看后端运行状态
- 查看后端日志

#### 数据库备份

- 配置 `sqlcmd` 路径（SQL Server 命令行工具）
- 配置备份输出目录
- 手动触发数据库备份
- 查看备份历史

#### Git 集成

- 配置 Git 仓库地址
- 从远程仓库拉取更新
- 查看 Git 历史记录
- 回滚到指定版本

## 7. 启动与验证

### 7.1 启动服务

#### 方式一：从源码运行

```powershell
cd D:\FinFlow
py -3.10 tools\finflow_manager.py
```

#### 方式二：运行 EXE

```powershell
cd D:\FinFlow\deploy\windows\dist
.\FinFlowManager.exe
```

### 7.2 验证部署

#### 后端验证

1. 打开浏览器访问：`http://127.0.0.1:8100/`
2. 检查是否显示 API 文档或返回 JSON 响应
3. 查看后端日志无报错

#### 前端验证

1. 在管理器中点击"打开前端"
2. 检查是否正常显示登录页面
3. 尝试登录系统
4. 检查主要菜单和列表页正常

#### 功能验证

- [ ] 首页正常打开
- [ ] 登录功能正常
- [ ] 主要菜单可访问
- [ ] 核心列表页正常加载
- [ ] 外部系统凭据可正常使用
- [ ] 数据库备份功能正常
- [ ] 托盘管理器状态正常

### 7.3 常见端口

| 服务 | 端口 | 说明 |
|------|------|------|
| 后端 API | 8100 | FastAPI 后端服务 |
| 前端 | 80/443 | Nginx/Apache（如果使用反向代理） |

## 8. 日常运维

### 8.1 查看日志

#### 后端日志

- 管理器 → "日志查看"页
- 或直接查看：`D:\FinFlow\backend\logs\`

#### 系统日志

- Windows 事件查看器 → 应用程序日志

### 8.2 数据库备份

#### 手动备份

1. 管理器 → "运维工具"页
2. 点击"立即备份"
3. 备份文件保存在配置的备份目录

#### 备份文件位置


### 8.3 服务重启

#### 通过管理器

1. 管理器 → "服务状态"页
2. 点击"重启后端"

#### 手动重启

```powershell
# 停止
cd D:\FinFlow
.venv\Scripts\python.exe tools\finflow_manager.py --stop

# 启动
.venv\Scripts\python.exe tools\finflow_manager.py
```

### 8.4 监控健康状态

- 管理器托盘图标：绿色表示正常
- "健康检查"按钮：显示当前状态
- 后端日志：检查是否有持续报错

## 9. 升级与回滚

### 9.1 代码升级

#### 方式一：Git 拉取

```powershell
cd D:\FinFlow
git pull
```

#### 方式二：导入发布包

1. 准备 ZIP 发布包，包含：
   ```
   backend\
   frontend\dist\
   tools\
   deploy\
   start_app.bat
   ```

2. 管理器 → "Git 拉取更新"页
3. 选择 ZIP 文件并导入

#### 升级后操作

```powershell
# 更新后端依赖
cd D:\FinFlow\backend
.venv\Scripts\pip.exe install -r requirements.txt

# 重启后端
cd D:\FinFlow
.venv\Scripts\python.exe tools\finflow_manager.py --restart
```

### 9.2 数据库升级

**重要**：升级前必须先备份数据库！

```powershell
# 备份数据库
cd D:\FinFlow
.venv\Scripts\python.exe tools\finflow_manager.py --backup
```

### 9.3 回滚操作

#### 代码回滚

```powershell
cd D:\FinFlow
git log --oneline -n 5
git checkout <上一个稳定版本的提交号>
```

#### 数据回滚

```powershell
# 使用 sqlcmd 恢复数据库
sqlcmd -S localhost -U admin -P your_password -d finflow -Q "RESTORE DATABASE finflow FROM DISK = 'D:\FinFlow\backups\finflow_backup_20240101_120000.bak' WITH REPLACE"
```

## 10. 常见问题

### 10.1 后端启动失败

**问题**：后端无法启动

**排查步骤**：

1. 检查 `.env` 文件是否存在
2. 检查 `DATABASE_URL` 或 `DB_*` 配置是否正确
3. 检查 `backend\.encryption.key` 是否存在
4. 查看后端日志文件
5. 运行语法检查

```powershell
cd D:\FinFlow\backend
.venv\Scripts\python.exe -m py_compile main.py
```

### 10.2 前端打开空白页

**问题**：前端页面空白或无法加载

**排查步骤**：

1. 检查 `frontend\dist` 目录是否存在
2. 检查 `VITE_API_BASE_URL` 是否为 `/api`
3. 检查后端服务是否正常运行
4. 查看浏览器控制台错误

### 10.3 登录后接口 401/403

**问题**：登录后访问接口返回 401 或 403

**排查步骤**：

1. 检查 `SECRET_KEY` 是否与发布前一致
2. 检查后端是否重启成功
3. 清除浏览器缓存和 cookie
4. 检查权限配置

### 10.4 外部系统凭据失效

**问题**：马克系统等外部系统凭据无法使用

**排查步骤**：

1. 检查 `backend\.encryption.key` 是否正确
2. 检查 `MARKI_USER` / `MARKI_PASSWORD` / `MARKI_SYSTEM_ID` 是否正确
3. 检查外部服务网络是否可达

### 10.5 数据库连接失败

**问题**：无法连接数据库

**排查步骤**：

1. 检查数据库服务是否运行
2. 检查数据库连接参数（主机、端口、用户名、密码）
3. 检查防火墙是否放行数据库端口
4. 检查 ODBC 驱动是否安装

### 10.6 托盘管理器无法启动

**问题**：管理器无法启动或立即退出

**排查步骤**：

1. 检查 Python 版本是否为 3.10
2. 检查依赖是否完整安装
3. 从源码运行查看错误信息
4. 检查是否有权限问题
